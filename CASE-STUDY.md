# Кейс: Модальное окно с валидацией и сохранением состояния

## 🎯 Проблема клиента
На сайте HTML нужно добавить попап-окно для перехода к бронированию
билетов через виджет Яндекс.Билетов. Требования:
- Пользователь должен дать согласие на обработку персональных данных (чекбокс).
- Кнопка перехода неактивна, пока согласие не получено.
- После согласия окно закрывается, происходит переход.
- Окно не должно показываться повторно при следующих визитах.

## 💡 Моё решение
Реализовал компонент на чистом Vanilla JS без библиотек. Ключевые
технические решения:

### 1. Валидация через атрибут `disabled`
Кнопка блокируется нативным HTML-атрибутом, а не через CSS-классы. Это
гарантирует доступность (accessibility) — скринридеры корректно
озвучивают состояние кнопки.

### 2. Сохранение состояния через `localStorage`
После первого согласия состояние сохраняется в браузере
пользователя. При повторных визитах попап не показывается — это
улучшает UX и снижает раздражение.

### 3. Закрытие по клавише `Esc`
Добавил обработку глобального события `keydown`. Пользователь может
закрыть окно без поиска кнопки "Закрыть" — это стандартное поведение
для десктопных пользователей.

### 4. Закрытие по клику вне окна
Реализовано через проверку `event.target === modalOverlay`. Это
защищает от случайных срабатываний при клике внутри самой модалки.

## 🛠 Технологии

- CSS3 (Flexbox для центрирования, `:disabled` для состояний)
- Vanilla JavaScript (ES6+, DOM API, localStorage, Event Listeners)
